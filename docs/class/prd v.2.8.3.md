## 1. 유저 스토리 10개

> 형식: "[CEO를 위한 AI 바이브코딩 스쿨의 고객]으로서, 나는 [이유]를 위해 [반응]을 원한다."
> 각 스토리는 prd v.1.1의 문제(P1~P5)·AX 유형(A/B/C)과 연결된다.

| ID | 유저 스토리 | 대응 문제/유형 | 연결 화면 |
|---|---|---|---|
| US-01 | **AI 도입을 검토 중인 비개발자 CEO(A유형)**로서, 나는 코딩을 해본 적 없는 내가 따라갈 수 있는 과정인지 지원 전에 판단하기 위해, 5분 자가진단과 수료생의 실제 결과물 미리보기를 원한다. | P1, P3 / A | /program **[U]** |
| US-02 | **최고위과정 여러 곳을 비교 중인 CEO(C유형)**로서, 나는 대학·언론사 과정 대신 이 과정을 선택할 근거를 찾기 위해, 커리큘럼·기간·산출물이 다른 과정과 무엇이 다른지 한 화면에서 확인하기를 원한다. | P1 / C | /program **[U]** |
| US-03 | **지원서를 제출한 예비 수강생**으로서, 나는 바쁜 일정 중 전형 결과를 놓치지 않기 위해, 로그인 없이 매직링크로 내 지원 상태(접수→심사→발표)를 조회하기를 원한다. | 등록 퍼널 | /apply/status **[U: P1→P0 승격]** |
| US-04 | **합격 후 등록 절차를 밟는 CEO**로서, 나는 결제·세금계산서 실무를 직접 처리할 시간이 없기 때문에, 인보이스 확인과 세금계산서 요청을 재무 담당 직원에게 위임하기를 원한다. | P4(위임) | /portal/billing **[U]** |
| US-05 | **실습 중 에러에 막힌 수강생 CEO**로서, 나는 그날 실습을 포기하고 뒤처지는 것을 막기 위해, 막힌 화면에서 바로 AI 조교에게 질문하고 해결이 안 되면 1:1 코칭·보충 세션으로 이어서 연결되기를 원한다. | P3 | /portal/ai **[U]** |
| US-06 | **파일럿이 흐지부지된 경험이 있는 CEO(B유형)**로서, 나는 이번에는 투자 대비 효과를 회사에 증명하기 위해, 내 결과물(Build)에 회사 적용 여부와 절감·개선 효과 메모를 기록하고 추적하기를 원한다. | P5 / B | /portal/builds **[U]** |
| US-07 | **CEO와 함께 수강하는 동반 임직원(assistant)**으로서, 나는 CEO와 조직 사이의 온도 차를 줄이고 같은 속도로 추진하기 위해, CEO와 동일한 세션 자료·과제를 보면서 CEO가 위임한 할 일만 따로 모아 처리하기를 원한다. | P4 | /portal/tasks **[N]** |
| US-08 | **출장으로 세션에 결석한 수강생 CEO**로서, 나는 한 번의 결석으로 흐름을 놓치지 않기 위해, 강의 녹화 영상과 영상 기반 Q&A에 더해 그 주차의 따라잡기 체크리스트를 받기를 원한다. | P3(완주) | /portal/sessions/[id] **[U]** |
| US-09 | **수료를 앞둔 수강생 CEO**로서, 나는 배운 것을 우리 회사의 AI 추진 계획으로 전환하기 위해, 10주차 발표용 AX 로드맵 초안을 과정 중 결과물·과제와 연결해 작성하기를 원한다. | P1, P5 | /portal/roadmap **[N]** |
| US-10 | **수료 후 6개월 지난 동문 CEO**로서, 나는 빠르게 낡는 AI 지식으로 다시 정보 격차가 생기는 것을 막기 위해, 멤버십으로 전 기수 녹화본·교재 아카이브와 의사결정 브리프를 계속 받아보기를 원한다. | P5, AS | /alumni/archive **[N]** |

## 2. 화면정의 — v2.7 7장 사이트맵 업그레이드

> 표기: **[U]** = 기존 화면 업그레이드 · **[N]** = 신규 화면 · (무표기) = v2.7 그대로 승계.
> 여기 없는 v2.7 라우트(/trends, /login, /portal/cohort, /portal/chat, /alumni/profile, /admin/* 등)는 전부 무변경 승계.

### 2.1 공개 영역

| 라우트 | 구분 | 정의 |
|---|---|---|
| /program | **[U]** | 기존 과정안내에 ① **5문항 자가진단**(비개발자 적합도, 결과에서 지원 CTA) ② **수료생 Build 미리보기** 3건(공개 동의분) ③ **"다른 과정과 무엇이 다른가" 비교 섹션**(경쟁사_비교분석2.md의 포지셔닝 채택안 — 최고위과정·플랫폼 강의·유튜브 대비 3축: 실습·CEO 의사결정·수료 후 지속) 추가. — US-01·02 |
| /apply/status | **[U]** | v2.7에서 "미구현·P1 백로그"였던 지원 상태 조회를 **P0로 승격**. 매직링크 진입, 접수→심사→발표 3단계 표시, 상태 전환 시 알림톡(T-01·02·03)과 연동. — US-03 |

### 2.2 수강생 포탈

| 라우트 | 구분 | 정의 |
|---|---|---|
| /portal/billing | **[U]** | 기존 인보이스·세금계산서 요청에 **결제 실무 위임** 추가: 인보이스 열람·세금계산서 요청 권한을 연결된 assistant에게 부여(RLS: 본인 + 연결 assistant + admin). 위임 상태 배지 표시. — US-04 |
| /portal/ai | **[U]** | 기존 AI 조교에 **막힘 에스컬레이션** 추가: 질문 시 현재 세션·과제 컨텍스트 자동 태깅(v2.7 D-25 Contextual Tutor 활용), AI 답변으로 해결 안 될 때 같은 화면에서 1:1 코칭 예약(/portal/coaching)·보충 세션 신청(/portal/supplement)으로 원클릭 연결. 미해결 태그는 강사 검수 큐(/admin/ai)에 우선 노출. — US-05 |
| /portal/builds | **[U]** | 기존 결과물 등록(모듈 I)에 **회사 적용 추적** 강화: 적용 여부(검토중/파일럿/적용완료) + 효과 메모(시간 절감·비용·매출 등 자유 기입) 필드 추가. 90일 성과설문(E-9)의 사전 데이터로 연결. v2.7에서 "배포 미확인" 표시된 화면이므로 구현 전 배포 여부 재확인 필요. — US-06 |
| /portal/sessions/[id] | **[U]** | 기존 세션 상세(영상·영상 Q&A·자료·출석)에 **결석 따라잡기 체크리스트** 추가: 결석자에게 영상 시청 → 자료 확인 → 과제 제출 → 막히면 AI 조교 4단계 체크리스트를 자동 노출, 완료 시 출석 보완 기록. — US-08 |
| /portal/tasks | **[N]** | assistant 전용 **위임 할 일** 화면. CEO가 과제·자료·일정에서 위임한 항목을 상태(대기/진행/완료)로 관리, 완료 시 CEO에게 알림. 신규 테이블 `delegated_tasks`(RLS: CEO 본인·연결 assistant·admin) 필요. — US-07 |
| /portal/roadmap | **[N]** | **AX 로드맵 초안 작성** 화면. v2.7 KPI "AX 로드맵 초안 작성률 ≥70%"의 실행 수단(전용 화면 부재였음). 템플릿 기반 단계 작성 + 내 Build·과제 결과물 연결 + 10주차 발표 모드(전체화면). — US-09 |

## 4. KPI 연결 (v2.7 10장 승계 + 보강)

- US-01·02 → 랜딩→지원 전환율 ≥8% (승계)
- US-05 → AI 튜터 1차 답변 성공률 ≥85% (승계) + **신규: 에스컬레이션 후 24시간 내 해소율 ≥90%**
- US-06 → 결과물 1개 이상 완성률 ≥80% (승계) + **신규: 결과물 회사 적용(파일럿 이상) 비율 ≥40% (확인 필요 — 17기 실측 후 목표 확정)**
- US-09 → AX 로드맵 초안 작성률 ≥70% (승계 — 전용 화면으로 달성 수단 확보)
- US-10 → 멤버십 가입률 ≥30%·갱신율 ≥60% (승계)
