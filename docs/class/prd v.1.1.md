## 1. 문제 — 타깃 고객의 핵심 불편

CEO는 AI 도입 압박은 크지만, 다음 5가지가 겹쳐 의사결정을 못 내리고 있다.

| # | 핵심 불편 | 상태 |
|---|---|---|
| P1 | 우리 회사에서 **무엇부터 시작할지 판단 기준이 없다** | 가설 (인터뷰 검증 예정) |
| P2 | IT 부서·외주사의 **견적·일정을 검증할 수 없어** 남의 판단에 의존한다 | 가설 |
| P3 | 특강·세미나를 들어도 **실습 경험이 없어 실무 지시로 이어지지 않는다** | 확인후보 |
| P4 | **임직원의 저항·온도 차**로 추진 동력이 생기지 않는다 | 가설 |
| P5 | **투자 대비 효과(ROI)를 숫자로 확신할 수 없어** 결재를 미룬다 | 가설 |

종합하면: **CEO의 최대 장벽은 기술이 아니라 '판단 기준의 부재'다** (종합 가설 1 — 인터뷰 5건 중 3건 이상 1순위 시 채택).

## 2. 타깃 — 기업의 AX 상황·유형별 구체화

주 타깃: 국내 중견·중소기업 CEO·임원(AI 도입 의사결정권자), 기수제 선발. 확인 사실: 17기 운영 실적, 18기 준비 중. AX(AI Transformation) 진행 상황에 따라 3개 유형으로 나눈다.

| 유형 | AX 상황 | 전형적 모습 | 주요 페인 | 우선순위 |
|---|---|---|---|---|
| **A. 미착수형** | AI 도입 논의만 있고 실행 전 | 제조·유통 중소기업. ChatGPT 개인 사용 수준. IT 전담 인력 0~2명 | P1(시작점), P3(교육 무용) | **1순위 타깃** |
| **B. 파일럿 정체형** | 일부 부서 파일럿 후 확산 실패 | 중견기업. 챗봇·자동화 PoC 경험 있으나 성과 불명, 흐지부지 | P4(조직 저항), P5(ROI) | 2순위 |
| **C. 위임 검증형** | 외주·IT 부서 주도로 진행 중 | 개발 조직 보유 또는 SI 계약 중. CEO는 보고만 받음 | P2(견적 검증 불가) | 2순위 |
| (비타깃) 대기업 전사 AX | 전담 조직·컨설팅펌 계약 | 컨설팅펌 프로그램이 적합 — 경쟁사_비교분석2.md 참조 | - | 제외 |

- 동반 수강 대상: 각 CEO가 지정하는 임직원 1~2인(assistant 역할) — 상품화 여부는 본인판단 후보, 18기 전 결정.
- 유형별 비중은 추정 — 17기 수강생 데이터로 실측 후 확정. (확인 필요)

## 3. 가치 — 경쟁 대안과 다른 우리만의 해결책

한 문장: **"AI를 아는 CEO가 아니라, AI로 만들어 본 CEO가 됩니다."**

| 경쟁 대안의 한계 | 우리의 해결책 | 겨냥하는 페인 |
|---|---|---|
| 대학·언론사 최고위과정 — 이론·특강 나열, 실습 부재 | 매 회차 누적되는 **단일 실습 커리큘럼**, 수료 기준 = 자기 회사 문제를 푼 결과물 | P1, P3 |
| 실무자용 플랫폼 강의 — CEO 맥락 부재 | 코딩 기술이 아닌 **CEO의 의사결정**(견적 검증·조직 적용·ROI 판단) 중심 설계 | P2, P5 |
| 유튜브·서적 — 체계·피드백·완주 동기 부재 | 기수제 오프라인 실습 + 교수 직강 피드백 + **AI 조교 상시 질의**(포털) | P3 |
| 신생 부트캠프 — 신뢰도·지속성 부재 | **KAIST AI대학원 교수 직강** + 수료 후에도 끊기지 않는 **동문 멤버십·포털** | P4, 지속성 |

차별화의 축은 3개: ① 만들어보는 경험(실습) ② CEO 눈높이의 판단력(의사결정) ③ 수료 후 지속(포털·동문). 이 중 ③은 경쟁 대안 전부가 비어 있는 지점이며, AI4CEO Portal이 그 실체다.

## 4. 핵심 흐름 — 고객 경험 4단계 (관심 → 등록 → 수강 → 졸업 후 AS)

| 단계 | 고객 경험 | 제품(포털) 지원 | 성공 판정 |
|---|---|---|---|
| **① 관심** | 포지셔닝 메시지(경쟁사_비교분석2.md 5안 중 채택안)를 접하고 랜딩에서 커리큘럼·수료생 결과물 확인 → 지원서 제출 | 랜딩 페이지 · `applications` 지원서 접수 | 지원서 제출 수 / 랜딩 방문 수 |
| **② 등록** | 선발 안내 → 인보이스 수령 → 입금 → 매직링크로 포털 첫 로그인 → 비밀번호 설정 | 관리자 승인 → `invoices` 발행 → 입금 확인 → `enrollments.status` paid→enrolled → 매직링크 온보딩 | 승인→온보딩 완주율 (v1 Must 2·3 슬라이스) |
| **③ 수강** | 매주 세션 참석 + 포털에서 자료 열람·과제 제출 + 막히면 AI 조교에 즉시 질문 (임직원 동반 시 함께) | `sessions` 회차·자료 · `assignments`/`submissions` · AI 조교(`ai_question_logs`, 인증+호출 제한 — v1 Must 1) | `weekly_active_learner` ≥ 90% (North Star) |
| **④ 졸업 후 AS** | 수료 후에도 동문 자격으로 포털 접속 — 자료 재열람, 동문 네트워크, 후속 기수 소식·재교육 안내 | `profiles.role` alumni 전환 · 동문 멤버십(E-11, 18기 수료 후 구현 — v1 Won't 유지) | 수료 90일 후 포털 재방문율 (확인 필요 — 측정 설계 필요) |

- ④는 경쟁 대안이 모두 비어 있는 차별화 구간이지만, v1의 범위 규율에 따라 **18기 수료(2026-11) 이후 구현**. v1.1에서는 제품 비전으로만 명시.
- ①의 랜딩·지원 퍼널 계측은 v1 Must 4(North Star 이벤트 로깅)에 `application.submitted` 이벤트 추가를 검토 (Day04 판단).

## 5. 성공 지표 (v1 승계 + 흐름별 보강)

> 원 프롬프트에 5번 항목이 명시되지 않아, v1의 성공 지표를 승계하고 4단계 흐름에 맞춰 보강했다.

- **North Star**: `weekly_active_learner` ≥ 90% (③ 수강 단계 — v1과 동일)
- **퍼널 지표(신규)**: ① 지원 전환율 → ② 온보딩 완주율 → ③ 주간 활동률 → ④ 수료 90일 재방문율 — 단계별 1개씩
- **구현 완료 기준**: v1의 4개 항목(AI 조교 보안·핵심 슬라이스·결제 manual·검증 게이트) 그대로 유지

## 6. 검증 계획 (v1.1의 가설 해소)

1. 18기 합격자 사전 인터뷰(최소 5건)로 P1~P5 가설 검증 — 질문지는 시장조사_타겟페인2.md의 "다음 행동" 열 기준
2. 경쟁 과정 최신 모집 요강 수집 → 가격 포지셔닝 확정 (경쟁사_비교분석2.md "(확인 필요)" 해소)
3. 포지셔닝 문장 5안 인터뷰 반응 테스트 → 상위 2개를 ① 관심 단계 랜딩 카피로 채택
4. 17기 데이터로 타깃 유형(A/B/C) 비중 실측 → 2절 표 확정
