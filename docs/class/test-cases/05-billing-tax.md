# 05. 결제 · 세금계산서 (BIL)

질문: "billing과 세금계산서는 API 키만 있으면 바로 구현 가능한가?" — **정확한 답: 아니오.**

## 어댑터 준비도 실사 (2026-07-19 코드 기준)

| 구성요소 | 인터페이스 | 실제 구현 | 키 넣으면 되는가 |
|---|---|---|---|
| 계좌이체(bank_transfer) | ✅ `lib/billing/provider.ts` | ✅ 안내·수동확인 플로우 | 키 불필요 — **현재 유일한 실동작 경로** |
| 스마트스토어 | ✅ 어댑터 enum 존재 | 링크 안내만 (수동 대조) | 해당 없음 (수동 프로세스) |
| Toss Payments | ✅ enum + payments 테이블 | ❌ **스텁** — 결제창 호출·webhook 라우트·승인 처리 코드 없음 | **아니오** — 키 + 약 2~3일 개발 필요 (webhook 엔드포인트, paid 자동전환, 알림) |
| 팝빌 세금계산서 | ✅ `lib/billing/tax-invoice.ts` (manual/popbill, mock 모드) | mock 시뮬레이션 로그만 | **아니오** — 키(LinkID/SecretKey) + 팝빌 SDK 연동·사업자 인증·테스트베드 검증 필요 |
| 이메일 알림 | — | ✅ nodemailer(Gmail SMTP) 실동작 | 이미 동작 |
| 알림톡(Solapi) | ✅ notifications 테이블 queued 기록 | ❌ 발송 no-op | **부분** — 키만으론 부족: 카카오 채널 + 템플릿 사전 검수(수일, 승인 후 수정불가) 선행 필요 |

## 결론 (의사결정용)

- **지금 당장 운영 가능한 경로**: 계좌이체 안내 + 관리자 수동 확인 (단, BIL-04/05 두 개를 먼저 실배선해야 함 — 각각 반나절 작업).
- **Toss**: 키 발급 후 webhook 라우트 + 승인검증 + 상태전환 구현 필요 (2~3일).
- **팝빌**: 키 + 사업자 인증 + 테스트베드 후 실발행 전환 (2~3일). mock 구조는 준비됨.
- **알림톡**: 기술보다 **카카오 템플릿 검수 리드타임**이 임계 경로 — 지금 신청해야 함.
