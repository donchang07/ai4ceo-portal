## 1. 유저 스토리 10개

> 고객 경험 4단계(관심 → 등록 → 수강 → 졸업 후 AS, prd v.1.1 4장)를 모두 덮도록 배치했다.

| ID | 단계 | 유저 스토리 | 대응 문제/유형 |
|---|---|---|---|
| US-01 | 관심 | **AI 도입을 검토 중이지만 코딩 경험이 없는 미착수형 CEO(A유형)**로서, 나는 지원했다가 못 따라가면 시간과 체면을 모두 잃을까 두렵기 때문에, 스쿨이 **지원 전에 5분 자가진단과 수료생의 실제 결과물을 보여주는 액션**을 취해주기 원한다. | P1, P3 / A |
| US-02 | 관심 | **대학·언론사 최고위과정과 비교 중인 위임 검증형 CEO(C유형)**로서, 나는 비슷해 보이는 과정들 사이에서 선택 근거를 스스로 만들 수 없기 때문에, 스쿨이 **커리큘럼·기간·산출물을 경쟁 과정과 한 화면에서 비교해 보여주는 액션**을 취해주기 원한다. | P1 / C |
| US-03 | 등록 | **지원서를 제출하고 결과를 기다리는 예비 수강생**으로서, 나는 바쁜 일정 속에 전형 안내 메일을 놓쳐 기회를 잃고 싶지 않기 때문에, 스쿨이 **로그인 없이 매직링크로 내 전형 상태(접수→심사→발표)를 조회하게 해주는 액션**을 취해주기 원한다. | 등록 퍼널 |
| US-04 | 등록 | **합격 후 등록 절차를 밟는 CEO**로서, 나는 결제·세금계산서 같은 실무를 직접 처리할 시간이 없기 때문에, 스쿨이 **인보이스 확인과 세금계산서 요청을 내 재무 담당 직원에게 위임하게 해주는 액션**을 취해주기 원한다. | P4(위임) |
| US-05 | 수강 | **실습 중 에러에 막힌 수강생 CEO**로서, 나는 한 번 막혀서 그날 실습을 포기하면 그대로 뒤처지기 때문에, 스쿨이 **막힌 화면에서 바로 AI 조교에게 질문하고, 해결이 안 되면 1:1 코칭·보충 세션까지 한 흐름으로 연결해주는 액션**을 취해주기 원한다. | P3 |
| US-06 | 수강 | **파일럿이 흐지부지된 경험이 있는 정체형 CEO(B유형)**로서, 나는 이번에는 투자 대비 효과를 회사에 숫자로 증명해야 하기 때문에, 스쿨이 **내 결과물(Build)의 회사 적용 여부와 절감·개선 효과를 기록·추적하게 해주는 액션**을 취해주기 원한다. | P5 / B |
| US-07 | 수강 | **CEO와 함께 수강하는 동반 임직원(assistant)**으로서, 나는 CEO와 조직 사이의 온도 차를 줄이고 같은 속도로 추진해야 하기 때문에, 스쿨이 **CEO와 동일한 자료·과제를 보되 CEO가 내게 위임한 할 일만 따로 모아 관리하게 해주는 액션**을 취해주기 원한다. | P4 |
| US-08 | 수강 | **출장으로 세션에 결석한 수강생 CEO**로서, 나는 한 번의 결석이 완주 실패로 이어지는 것을 막아야 하기 때문에, 스쿨이 **녹화 영상·영상 Q&A에 더해 그 주차의 따라잡기 체크리스트를 제공하는 액션**을 취해주기 원한다. | P3(완주) |
| US-09 | 수강→수료 | **수료를 앞둔 수강생 CEO**로서, 나는 배운 것을 우리 회사의 AI 추진 계획으로 전환해야 하기 때문에, 스쿨이 **과정 중 결과물·과제를 연결해 10주차 발표용 AX 로드맵 초안을 작성하게 해주는 액션**을 취해주기 원한다. | P1, P5 |
| US-10 | 졸업 후 AS | **수료 후 6개월이 지난 동문 CEO**로서, 나는 빠르게 낡는 AI 지식 탓에 다시 정보 격차가 생기는 것이 두렵기 때문에, 스쿨이 **멤버십으로 전 기수 녹화본·교재 아카이브와 의사결정 브리프를 계속 제공하는 액션**을 취해주기 원한다. | P5, AS |

## 2. 화면정의

> 표기: **[U]** = v2.7 기존 화면 업그레이드 · **[N]** = 신규 화면.
> 각 화면은 ① 개요 ② 화면 구성 ③ 상태·엣지케이스 ④ 데이터·권한 순으로 정의한다.

### SCR-01. /program — 과정 안내 **[U]** (US-01 · US-02)

**① 개요**
- 목적: "따라갈 수 있을까"(US-01)와 "다른 과정과 뭐가 다른가"(US-02)라는 지원 전 2대 질문을 이 화면 안에서 해소하고 지원 CTA로 보낸다.
- 사용자/진입: 비로그인 방문자. 광고·검색·소개 유입.

**② 화면 구성** (위→아래)
1. 히어로 — 포지셔닝 문장(경쟁사_비교분석2.md 채택안) + 기수 모집 상태 배지 + 지원 CTA
2. 커리큘럼 개요 — 주차별 요약, 수료 기준(= 자기 회사 문제를 푼 결과물) 명시
3. **자가진단(신규)** — 5문항(코딩 경험·주당 가용 시간·AX 상황 A/B/C 판별 등), 완료 시 "비개발자 적합도" 결과 카드 + 결과별 맞춤 문구 + 지원 CTA
4. **수료생 Build 미리보기(신규)** — 공개 동의된 결과물 3건 카드(업종·문제·결과물 스크린샷·제작 주차)
5. **경쟁 비교 섹션(신규)** — 최고위과정·플랫폼 강의·유튜브 대비 3축(실습·CEO 의사결정·수료 후 지속) 비교표
6. FAQ + 하단 지원 CTA

**③ 상태·엣지케이스**
- 모집 마감 시: CTA를 "다음 기수 알림 신청"으로 전환
- 자가진단 중도 이탈 시: 응답 저장하지 않음(비로그인, 개인정보 미수집 원칙)
- Build 미리보기는 공개 동의분이 3건 미만이면 섹션 자체를 숨김(빈 카드 노출 금지)

**④ 데이터·권한**
- 자가진단 문항·결과 문구: 정적 콘텐츠(추후 CMS화 검토). 응답은 비저장, 결과 유형만 `application.submitted` 이벤트 속성으로 전달(퍼널 분석용)
- Build 미리보기: `builds` 중 `public_consent = true`만 노출

### SCR-02. /apply/status — 전형 상태 조회 **[U: P1→P0 승격]** (US-03)

**① 개요**
- 목적: 로그인 없이 매직링크로 전형 진행 상황을 확인시켜 이탈·불안을 줄인다.
- 사용자/진입: 지원서 제출자. 접수 확인 알림톡·메일의 매직링크로만 진입.

**② 화면 구성**
1. 지원자 이름·기수 확인 헤더
2. 3단계 스테퍼 — 접수완료 → 심사중 → 결과발표(합격/불합격/대기)
3. 단계별 안내문 — 현재 단계 설명 + 다음 단계 예상 일정
4. 합격 시: 등록 안내(입금·일정) 요약 + 포털 온보딩 링크 노출

**③ 상태·엣지케이스**
- 매직링크 만료(기본 7일): 만료 안내 + "새 링크 받기"(등록된 연락처로 재발송)
- 잘못된/위조 토큰: 상태 정보 일절 노출 없이 안내 페이지로
- 상태 전환 시 알림톡(T-01·02·03)과 본 화면 표시가 항상 일치해야 함(전환 트리거는 /admin/applications)

**④ 데이터·권한**
- `applications.status` 읽기 전용. 토큰은 서명된 일회성 매직링크(지원 건 단위), 개인정보는 이름 외 미노출

### SCR-03. /portal/billing — 인보이스·결제 위임 **[U]** (US-04)

**① 개요**
- 목적: CEO가 결제 실무(인보이스 확인·세금계산서 요청)를 연결된 assistant에게 넘기고 진행 상태만 확인하게 한다.
- 사용자/진입: 합격 CEO(본인) 및 위임받은 assistant. 포털 메뉴 또는 등록 안내 링크.

**② 화면 구성**
1. 인보이스 카드 — 금액·입금 계좌·기한·입금 상태(대기/확인중/완료)
2. 세금계산서 요청 폼 — 사업자 정보 입력, 요청 상태 표시
3. **위임 패널(신규)** — 연결된 assistant 목록 + "결제 실무 위임" 토글, 위임 중이면 화면 상단에 위임 상태 배지("○○님에게 위임됨")
4. 처리 이력 — 누가(본인/assistant) 언제 무엇을 했는지 타임라인

**③ 상태·엣지케이스**
- assistant 미연결 상태에서 위임 시도: 동반 임직원 연결 안내로 유도
- 위임 해제 시: assistant 화면에서 즉시 접근 차단, 진행 중 요청은 CEO에게 승계 알림
- 입금 확인은 manual 운영(v1 Must 3 승계) — 화면은 상태 표시만, 확인 처리는 admin

**④ 데이터·권한**
- `invoices` · 세금계산서 요청 테이블. RLS: 본인 + 위임 flag가 켜진 연결 assistant + admin
- 위임 상태 변경은 CEO만 가능, 모든 처리에 actor 기록

### SCR-04. /portal/ai — AI 조교 + 막힘 에스컬레이션 **[U]** (US-05)

**① 개요**
- 목적: 실습 막힘을 "AI 즉답 → 안 되면 사람"의 한 흐름으로 해소해 그날의 포기를 막는다. 완주율(North Star) 직결 화면.
- 사용자/진입: 수강생·assistant. 포털 전역의 "막혔어요" 버튼(세션·과제 화면에서 컨텍스트를 갖고 진입) 또는 메뉴.

**② 화면 구성**
1. 질문 입력부 — 진입 시 현재 세션·과제 컨텍스트 자동 태깅(v2.7 D-25 Contextual Tutor), 에러 메시지·스크린샷 첨부
2. AI 답변 스레드 — 대화형, 코드 블록 복사 지원
3. **해결 확인 바(신규)** — 답변마다 "해결됐어요 / 아직 안 돼요" 선택
4. **에스컬레이션 패널(신규)** — "아직 안 돼요" 2회 또는 사용자 직접 요청 시 노출: 1:1 코칭 예약(/portal/coaching)·보충 세션 신청(/portal/supplement) 원클릭 연결, 대화 스레드 자동 첨부
5. 내 질문 이력 — 해결/미해결 필터

**③ 상태·엣지케이스**
- 호출 제한 도달(v1 Must 1 승계): 제한 안내 + 에스컬레이션 경로는 항상 열어둠(질문 못 해서 포기하는 상황 방지)
- 미해결 태그 질문은 강사 검수 큐(/admin/ai)에 우선 노출, 24시간 내 해소 목표(KPI: 에스컬레이션 후 24h 해소율 ≥90%)
- AI 답변 실패(타임아웃 등): 재시도 + 바로 에스컬레이션 제안

**④ 데이터·권한**
- `ai_question_logs`(+ 해결 여부·에스컬레이션 상태 컬럼). RLS: 본인 + 연결 관계(CEO↔assistant는 서로 열람 가능 여부 — 기본 비공개, 옵션 검토) + admin
- 인증 필수·호출 제한(v1 Must 1)

### SCR-05. /portal/builds — 결과물 등록·회사 적용 추적 **[U]** (US-06)

**① 개요**
- 목적: 결과물을 "만들었다"에서 끝내지 않고 회사 적용과 효과까지 기록하게 해, CEO가 ROI를 스스로 증명할 데이터를 쌓는다.
- 사용자/진입: 수강생 CEO(작성), assistant(열람·초안 보조). 포털 메뉴·과제 완료 흐름에서 진입.

**② 화면 구성**
1. 내 Build 목록 — 카드(제목·스크린샷·주차·적용 상태 배지)
2. Build 등록/편집 — 제목·설명·스크린샷·데모 링크·관련 주차
3. **적용 추적 패널(신규)** — 적용 상태(검토중/파일럿/적용완료) + 효과 메모(시간 절감·비용·매출 등 자유 기입) + 상태 변경 이력
4. 공개 동의 토글 — /program 미리보기(SCR-01)·수료 발표 노출 동의
5. 로드맵 연결 표시 — 이 Build를 참조하는 AX 로드맵(SCR-08) 항목 링크

**③ 상태·엣지케이스**
- 적용 상태·효과 메모는 90일 성과설문(E-9)의 사전 데이터로 연결 — 수료 후에도 편집 가능(동문 권한)
- v2.7에서 모듈 I "배포 미확인" 표시 — 구현 전 배포 여부 재확인 선행(v2.8.3 승계)
- 공개 동의 철회 시 /program 노출 즉시 제거

**④ 데이터·권한**
- `builds`(+ `apply_status`, `effect_memo`, `public_consent` 컬럼). RLS: 본인 + 연결 assistant(열람) + admin
- KPI: 결과물 1개 이상 완성률 ≥80%(승계) · 파일럿 이상 적용 비율 ≥40%(17기 실측 후 확정)

### SCR-06. /portal/tasks — 위임 할 일 (assistant 전용) **[N]** (US-07)

**① 개요**
- 목적: 동반 임직원이 CEO에게 위임받은 일만 한곳에서 처리하게 해, CEO-조직 간 온도 차(P4)를 실행 단위로 줄인다.
- 사용자/진입: assistant(관리), CEO(위임 생성·진행 확인). 포털 메뉴 + 과제·자료·일정 화면의 "위임하기" 버튼.

**② 화면 구성**
1. (assistant 뷰) 위임받은 할 일 목록 — 상태(대기/진행/완료) 칸반 또는 리스트, 출처(어느 과제·자료·일정에서 위임됐는지) 링크
2. (CEO 뷰) 내가 위임한 항목 목록 — 진행 상태 요약, 재촉·회수 버튼
3. 할 일 상세 — 메모·마감일·완료 처리, 완료 시 CEO에게 알림
4. 빈 상태 — "아직 위임받은 할 일이 없습니다" + 위임 흐름 안내

**③ 상태·엣지케이스**
- CEO-assistant 연결 해제 시: 미완료 위임 건은 CEO에게 반환 표시
- 결제 위임(SCR-03)은 권한 위임이므로 이 화면의 할 일 위임과 구분 — 여기서는 실행 업무만 다룸
- 위임은 연결된 CEO↔assistant 사이에서만 생성 가능

**④ 데이터·권한**
- 신규 테이블 `delegated_tasks`(발신 CEO, 수신 assistant, 출처 엔티티, 상태, 마감일). RLS: 해당 CEO·연결 assistant·admin

### SCR-07. /portal/sessions/[id] — 세션 상세 + 결석 따라잡기 **[U]** (US-08)

**① 개요**
- 목적: 결석 1회가 이탈로 번지지 않도록, 세션 상세 안에서 따라잡기를 완결시킨다.
- 사용자/진입: 수강생·assistant. 세션 목록·알림톡 링크.

**② 화면 구성**
1. 세션 헤더 — 주차·제목·일시·출석 상태(출석/결석/보완완료)
2. 녹화 영상 + 영상 기반 Q&A(v2.7 승계)
3. 세션 자료·과제 링크
4. **따라잡기 체크리스트(신규, 결석자에게만 자동 노출)** — ① 영상 시청 → ② 자료 확인 → ③ 과제 제출 → ④ 막히면 AI 조교(SCR-04로 컨텍스트 연결) 4단계, 각 단계 완료 체크
5. 체크리스트 완료 시 — 출석 보완 기록 + 완료 배지

**③ 상태·엣지케이스**
- 녹화 업로드 전 결석자 진입: "영상 준비 중" 표시 + 업로드 시 알림
- 체크리스트 진행 중 다음 세션 도래: 미완료 상태 유지하되 최신 세션 우선 안내
- 영상 시청 완료 판정 기준(시청률 임계값)은 운영 정책으로 확정 필요 (확인 필요)

**④ 데이터·권한**
- `sessions` · 출석 테이블(+ 보완 기록), 체크리스트 진행 상태 저장. RLS: 수강생 본인 진행 상태 + admin

### SCR-08. /portal/roadmap — AX 로드맵 작성 **[N]** (US-09)

**① 개요**
- 목적: KPI "AX 로드맵 초안 작성률 ≥70%"의 실행 수단. 과정 중 산출물을 회사의 AI 추진 계획으로 전환한다.
- 사용자/진입: 수강생 CEO. 포털 메뉴, 8주차 이후 홈 배너로 유도.

**② 화면 구성**
1. 템플릿 기반 단계 작성 — 현황 진단 → 우선 과제 → 90일 계획 → 확산 계획 (각 단계 가이드 문구 포함)
2. Build·과제 연결 — 내 결과물(SCR-05)·과제 제출물을 로드맵 항목에 근거로 첨부
3. 저장·버전 — 자동 저장, 초안/발표본 구분
4. **발표 모드** — 10주차 발표용 전체화면 뷰(목차 네비게이션 포함)
5. 내보내기 — PDF 다운로드 (확인 필요 — M3 범위 내 포함 여부)

**③ 상태·엣지케이스**
- 빈 상태: 템플릿 예시(익명화된 샘플 로드맵) 미리보기 제공
- Build 미등록 상태에서 연결 시도: SCR-05 등록 흐름으로 유도
- 수료 후: 동문 권한으로 열람·편집 유지(회사 적용의 연속성)

**④ 데이터·권한**
- 신규 테이블 `roadmaps`(단계별 콘텐츠, 연결 Build/과제 참조, 버전). RLS: 본인 + 연결 assistant(열람) + admin

### SCR-09. /alumni/archive — 동문 아카이브 **[N]** (US-10)

**① 개요**
- 목적: 수료 후 지식 노후화(P5)를 막는 멤버십 핵심 혜택. 경쟁 대안이 전부 비어 있는 "수료 후 지속" 축의 실체.
- 사용자/진입: 멤버십 active 동문. 동문 홈·멤버십 안내에서 진입.

**② 화면 구성**
1. 탐색 필터 — 기수·주차·주제별 녹화본·교재 브라우징
2. 콘텐츠 상세 — 영상 재생·교재 다운로드
3. 의사결정 브리프 구독 진입점 — /trends 통합분 안내·구독 설정
4. 최신 기수 안내 배너 — 진행 중 기수 콘텐츠는 종료 후 공개 정책 표시

**③ 상태·엣지케이스**
- 멤버십 만료: 목록은 보이되 재생·다운로드 차단 + 갱신 유도(/alumni/membership 연동)
- 진행 중 기수 콘텐츠 접근 시: "기수 종료 후 공개" 안내
- 저작권·초상권: 녹화본 공개 범위는 출연 수강생 동의 정책 선행 (확인 필요)

**④ 데이터·권한**
- 콘텐츠 메타(기수·주차·유형) + 멤버십 상태 조인. RLS: `profiles.role = alumni` AND 멤버십 active + admin
- KPI: 멤버십 가입률 ≥30% · 갱신율 ≥60%(승계)
