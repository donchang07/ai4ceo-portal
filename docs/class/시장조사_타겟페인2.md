# 시장조사 — 타겟 페인 분류 2

- 원본: [시장조사요약서2.md](시장조사요약서2.md)
- 작성일: 2026-07-20
- 분류 기준
  - **가설**: AI의 추측. 고객 인터뷰·데이터로 검증하기 전에는 사실로 취급하지 않는다
  - **확인후보**: 조사·인터뷰로 비교적 쉽게 사실 여부를 확인할 수 있는 항목
  - **본인판단 후보**: 검증보다는 운영자(본인)의 전략적 의사결정이 필요한 항목

## 분류표

| AI의 답 | 분류 | 다음 행동 |
|---|---|---|
| CEO는 AI 적용의 시작점을 판단할 기준이 없어 막막해한다 | 가설 | 예비 고객 인터뷰 5건에서 "AI 도입의 가장 큰 장벽"을 개방형으로 질문해 시작점 부재가 상위에 오르는지 확인 |
| 현재 대안은 벤치마킹·컨설팅·"ChatGPT 계정 지급" 수준에 머문다 | 확인후보 | 인터뷰에서 "지금까지 실제로 해본 것"을 구체적으로 물어 대안 목록과 빈도 수집 |
| CEO는 IT 부서·외주사의 견적과 일정을 검증하지 못해 불안하다 | 가설 | 인터뷰에서 견적·일정 불신을 겪은 실제 사례가 있는지, 그때 어떻게 대응했는지 확인 |
| 직접 만들어 보는 경험이 기술 불신 해소에 효과적이다 | 본인판단 후보 | 본 과정의 핵심 가치제안이므로 파일럿 기수 전후 설문으로 효과를 측정하되, 커리큘럼 방향은 운영자가 결정 |
| 기존 AI 특강·세미나는 개념 소개에 그쳐 실무로 이어지지 않는다 | 확인후보 | 타깃 CEO의 기존 교육 수강 이력과 불만 지점(이론 위주/난이도/시간)을 인터뷰로 확인 |
| CEO도 직접 실습에 시간을 낼 의사가 있다 | 가설 | 인터뷰에서 주당 가용 시간과 "직접 코딩 실습" 참여 의향을 확인 — 부정되면 과정 설계 자체를 재검토 |
| 임직원의 저항("일자리 축소", "또 새 도구")이 도입 동력을 꺾는다 | 가설 | 인터뷰에서 조직 저항의 실제 경험 여부와, 도입 실패의 주원인이 저항인지 CEO 추진력 부족인지 확인 |
| CEO·임직원 동반 수강이 구매 결정의 차별화 요인이 된다 | 본인판단 후보 | 수요는 인터뷰로 참고하되, 동반 수강을 상품 구조에 넣을지는 가격·운영 부담을 고려해 운영자가 결정 |
| AI 투자 ROI를 숫자로 확신할 수 없어 결재를 미룬다 | 가설 | 인터뷰에서 최근 AI 관련 지출 의사결정 사례와 그때 요구한 근거(정량 ROI/타사 사례)를 확인 |
| 교육 수강 결정에는 ROI보다 네트워킹·평판이 더 크게 작용할 수 있다 | 확인후보 | 인터뷰에서 최고경영자 과정 등 기존 교육을 선택한 실제 이유를 물어 구매 동기 순위 확인 |
| 소액 파일럿·타사 사례 인용이 현재의 ROI 대응 방식이다 | 확인후보 | 인터뷰에서 파일럿 경험 유무와 결과, 내부 설득에 쓴 자료를 확인 |
| CEO의 최대 장벽은 기술이 아니라 판단 기준의 부재다 (종합 가설 1) | 가설 | 위 개별 확인 결과를 종합해 판단 — 인터뷰 5건 중 3건 이상에서 시작점·검증 문제가 1순위로 나오면 채택 |
| 실습 부재가 기존 교육 불만의 핵심이다 (종합 가설 2) | 가설 | 교육 경험 인터뷰 결과로 검증 — 채택 시 "직접 만든다"를 마케팅 핵심 메시지로 확정 |
| 임직원 동반 수강이 차별화 요인이다 (종합 가설 3) | 본인판단 후보 | 수요 확인 결과와 무관하게 운영 리소스·수익 구조를 따져 포함 여부를 운영자가 최종 결정 |

## 요약

- **가설 7건**: 전부 고객 인터뷰(최소 5건)로 검증 필요 — 인터뷰 질문지에 반영
- **확인후보 4건**: 인터뷰에서 "실제 경험"을 묻는 폐쇄형·구체형 질문으로 확인
- **본인판단 후보 3건**: 커리큘럼 방향, 동반 수강 상품화 등 — 검증 데이터는 참고 자료로만 쓰고 운영자가 결정
