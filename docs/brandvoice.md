## 확정 정보

- **서비스 이름:** AI4CEO Portal
- **한 줄 무드:** 바쁜 CEO가 AI를 막연한 유행이 아니라 회사에 적용할 실행 과제로 받아들이게 해주는, 조용하지만 단단한 디지털 운영실.
- **브랜드 역할:** 강의 플랫폼이 아니라, CEO가 배우고 만들고 묻고 적용하는 전 과정을 정리해주는 AI 실행 파트너.

## 1. Voice Summary

Sounds like a calm executive advisor who understands AI deeply, explains it without showing off, and helps a CEO move from curiosity to company-level action.

AI4CEO Portal은 과장하지 않는다. 대신 "이번 주에 무엇을 보고, 무엇을 만들고, 어디에 적용할지"를 차분하게 정리해준다.

## 2. Core Personality Traits

**신뢰감 있는**
- We are: CEO가 민감한 회사 정보와 학습 과정을 맡길 수 있도록 근거, 권한, 공개 범위를 분명히 말한다.
- We are not: 권위를 과시하거나 "모르면 뒤처진다"며 불안을 자극하지 않는다.
- This sounds like: "이 영상은 해당 기수 수강생에게만 read-only로 공개됩니다."
- This does NOT sound like: "AI 시대에 뒤처지지 않으려면 지금 당장 봐야 합니다."

**실행 중심의**
- We are: 배움보다 다음 행동을 먼저 보여준다. 강의, 과제, Drive 파일, AI 질문은 모두 결과물 완성으로 연결된다.
- We are not: 기능을 많이 나열하거나 학습량 자체를 성과처럼 포장하지 않는다.
- This sounds like: "다음 세션 전까지 Build Brief 초안을 작성해 주세요."
- This does NOT sound like: "다양한 기능을 마음껏 둘러보세요."

**쉽게 풀어주는**
- We are: AI, vibe coding, Claude Code 같은 용어를 CEO가 판단하고 지시할 수 있는 언어로 번역한다.
- We are not: 개발자용 전문어를 그대로 던지거나, 너무 가볍게 단순화하지 않는다.
- This sounds like: "이 파일은 대화방 참여자가 함께 수정할 수 있는 공유 자료입니다."
- This does NOT sound like: "cohort_rw permission이 active 상태입니다."

**절제된 자신감**
- We are: 할 수 있는 것과 아직 확인이 필요한 것을 구분해서 말한다.
- We are not: AI가 모든 답을 안다는 식으로 말하지 않는다.
- This sounds like: "이 답변은 4주차 강의 영상과 과제 설명을 기준으로 작성되었습니다."
- This does NOT sound like: "정답은 이것입니다."

**따뜻한 운영감**
- We are: CEO가 컴퓨터를 잘 다루지 못해도 민망하지 않게 안내한다.
- We are not: 실수나 지연을 혼내거나, 젊은 개발자 문화처럼 장난스럽게 몰아가지 않는다.
- This sounds like: "오프라인 보충 세션에서 함께 따라가실 수 있습니다."
- This does NOT sound like: "아직도 설정을 못 하셨나요?"

## 4. Vocabulary Guide

### USE

| 표현 | 쓰는 이유 |
|---|---|
| 기수 | 과정 운영의 기본 단위. CEO 교육의 폐쇄성과 맥락을 살린다. |
| 다음 행동 | 수강생이 지금 해야 할 일을 명확히 만든다. |
| 결과물 | 출석보다 중요한 성과 기준을 드러낸다. |
| 회사 적용 | AI 학습을 경영 실행으로 연결한다. |
| Build Brief | 만들기 전 문제 정의를 가볍고 명확하게 부른다. |
| 공유 자료 | Google Drive 파일을 사용자가 이해하기 쉬운 말로 설명한다. |
| 강의 다시보기 | "녹화본"보다 수강생 친화적인 표현. |
| 참고한 자료 | AI 답변의 근거와 신뢰를 보여준다. |
| 공개 범위 | 프로필, 파일, 영상 권한을 설명할 때 핵심 표현. |
| 보충 세션 | 컴퓨터 사용이 익숙하지 않은 수강생에게 부담 없는 표현. |
| 운영자 확인 | 입금, 권한, 파일 문제에서 신뢰감을 준다. |
| 변경 이력 | 기수별 커리큘럼 변화의 투명성을 보여준다. |
| read-only / 읽기 전용 | 기술 정책은 필요할 때 한글 설명과 함께 쓴다. |
| 함께 수정 | read/write보다 사용자에게 직관적이다. |
| 지금 확인하기 | CTA에서 강요 없이 행동을 유도한다. |

### AVOID

| 표현 | 피하는 이유 |
|---|---|
| 혁명, 대전환, 무조건 | AI 과장처럼 들린다. |
| 뒤처지지 않으려면 | CEO의 불안을 자극한다. |
| 초간단, 누구나 5분 만에 | 실제 학습 난이도를 가볍게 만든다. |
| 마스터, 정복 | 교육 서비스의 과장된 성공 약속처럼 보인다. |
| 닥치고 실행 | 대상 사용자와 맞지 않는 거친 톤이다. |
| 개발자만 아는 약어 단독 사용 | CEO와 assistant에게 장벽이 된다. |
| 실패했습니다 | 대체 행동 없이 차갑게 느껴진다. |
| 오류입니다 | 원인과 다음 조치를 함께 제시해야 한다. |
| 자유롭게 둘러보세요 | 바쁜 사용자에게 방향을 주지 못한다. |
| 커뮤니티에 참여하세요 | CEO에게는 목적 없는 소셜처럼 들릴 수 있다. |
| AI가 알아서 해드립니다 | 책임 범위가 불명확하다. |
| 공개됩니다 | 공개 범위를 함께 말하지 않으면 불안하다. |
| 빨리 제출하세요 | 독촉처럼 느껴진다. |
| 혜택을 놓치지 마세요 | 고가 CEO 과정과 어울리지 않는 판촉 톤이다. |

## 5. Tone Adaptation Rules

| 맥락 | 톤 조절 | 예시 문구 |
|------|---------|-----------|
| 온보딩 / 첫 화면 | 따뜻하지만 바로 실행 가능한 방향 제시 | "이번 기수의 다음 일정과 오늘 할 일을 정리했습니다." |
| Cohort Home | 요약형, 판단 중심, 불필요한 설명 최소화 | "다음 세션까지 과제 1건과 새 강의 영상 1건이 남아 있습니다." |
| 빈 상태(empty state) | 사용자를 탓하지 않고 다음 입력을 제안 | "아직 공유된 파일이 없습니다. 대화방에서 자료를 올리면 여기에 모입니다." |
| 성공/완료 피드백 | 조용한 인정, 다음 단계 연결 | "제출되었습니다. 운영자가 확인하면 피드백을 알려드립니다." |
| 에러/안내 | 원인보다 해결 행동을 먼저 제시 | "Drive 권한을 확인하지 못했습니다. 운영자에게 다시 동기화를 요청할 수 있습니다." |
| 푸시/알림톡 | 짧고 명확하게, 독촉보다 안내 | "#{이름}님, #{과제명} 마감이 내일입니다. 포탈에서 제출 상태를 확인해 주세요." |
| AI 답변 | 출처, 불확실성, 다음 행동을 포함 | "이 답변은 3주차 영상 요약과 과제 설명을 기준으로 작성했습니다." |
| 랜딩/소개 페이지 | 신뢰, 결과물, 운영 모델을 선명하게 | "AI를 배우는 데서 끝내지 않고, 회사에 적용할 첫 결과물을 만듭니다." |
| 관리자 화면 | 감정 없는 운영 언어, 상태와 조치 중심 | "영상 2건의 Drive 권한 동기화가 필요합니다." |

## 6. Style Rules

- 존댓말을 기본으로 한다. "~합니다"를 기본으로 쓰되, 화면 안의 짧은 안내는 "~해 주세요"를 쓴다.
- 문장은 짧게 쓴다. 한 문장에 하나의 행동만 담는다.
- 이모지는 기본적으로 쓰지 않는다. 축하/수료 맥락에서도 필요하면 최대 1개만 쓴다.
- 영어 기능명은 필요한 경우만 쓴다. 처음 등장할 때는 한글 설명을 붙인다.
- AI 답변은 출처와 한계를 함께 말한다. 확정되지 않은 내용은 확정처럼 쓰지 않는다.
- 공개/권한/파일/영상 관련 문구는 항상 누가 볼 수 있는지, 누가 수정할 수 있는지 함께 말한다.
- CTA는 명령보다 확인/진행형을 쓴다. 예: "제출하기", "영상 보기", "권한 확인하기".
- 경고 문구는 차갑게 끝내지 않는다. 항상 복구 행동을 붙인다.

## 7. Do's and Don'ts

### Do

- 사용자에게 오늘 해야 할 일을 먼저 보여준다.
- "강의 수강"보다 "결과물 완성"과 "회사 적용"을 강조한다.
- AI 답변에는 가능한 출처를 남긴다.
- 권한, 공개 범위, read-only/read-write 정책을 명확히 말한다.
- 컴퓨터 사용이 익숙하지 않은 사용자를 배려하는 표현을 쓴다.
- 관리자 화면은 상태, 원인, 조치를 분리해 보여준다.
- 고가 CEO 프로그램에 맞는 조용한 자신감을 유지한다.

### Don't

- AI 유행어와 과장된 공포 마케팅을 쓰지 않는다.
- 수강생이 늦거나 막혔을 때 다그치지 않는다.
- 기능을 많이 보여주는 것을 좋은 경험으로 착각하지 않는다.
- 커뮤니티를 목적 없는 친목 공간처럼 말하지 않는다.
- AI가 법률/경영/기술 결정을 대신 내리는 것처럼 표현하지 않는다.
- 공개 프로필을 은근히 강요하지 않는다.
- 영어 약어만으로 핵심 기능을 설명하지 않는다.

## Sample Copy

### Landing

- "AI를 배우는 데서 끝내지 않고, 우리 회사에 적용할 첫 결과물을 만듭니다."
- "10주 동안 배우고, 만들고, 묻고, 다시 적용합니다."
- "CEO와 임원을 위한 기수제 AI 실행 포탈."

### Cohort Home

- "이번 주에 확인할 항목을 정리했습니다."
- "새 강의 영상 1건, 미제출 과제 1건이 있습니다."
- "공유 Drive 파일은 기수 참여자가 함께 수정할 수 있습니다."

### AI Tutor

- "이 답변은 5주차 강의 영상과 과제 설명을 기준으로 작성했습니다."
- "현재 자료만으로는 확정하기 어렵습니다. 관련 Drive 파일을 추가하면 더 정확히 답변할 수 있습니다."
- "이 질문은 강사 검수가 필요할 수 있습니다."

### Permission / Privacy

- "이 영상은 해당 기수 수강생과 연결된 assistant에게만 읽기 전용으로 공개됩니다."
- "이 프로필은 기본 비공개입니다. 원하실 때만 공개 범위를 바꿀 수 있습니다."
- "대화방 공유 파일은 참여자가 함께 수정할 수 있습니다."

### Completion

- "수료 처리가 완료되었습니다. 결과물과 수료 정보는 포탈에서 확인하실 수 있습니다."
- "프로필 작성은 선택 사항입니다. 아무것도 공개하지 않아도 서비스 이용에 제한은 없습니다."
